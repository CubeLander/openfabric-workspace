一、项目概述
1.1 项目背景
为提升国产边缘端AI芯片大模型推理能力，充分挖掘睿芯芯片算力资源，解决大语言模型在边缘硬件上推理延迟高、吞吐低、显存占用大等问题。本项目基于睿芯芯片硬件平台，完成Qwen3-8B大语言模型串联部署，针对推理链路进行深度优化，搭建稳定、高效、可落地的边缘大模型推理方案，满足智能交互、嵌入式AI、离线推理等业务场景需求。
1.2 项目研究范围
本项目针对睿芯芯片硬件架构，完成Qwen3-8B模型移植、模型切分、多芯片串联调度。重点开展KV Cache优化、推理解码优化、内存调度优化、算子适配优化等技术研发，完成模型工程化部署与性能调优。
1.3 项目总体目标
（1）功能目标：实现Qwen3-8B模型在睿芯多芯片硬件平台稳定串联运行，模型加载正常、文本生成流畅，上下文长度支持2048Token；
（2）性能目标：模型生成吞吐率不低于25 token/s，首token生成延迟（TTFT）控制在2s以内；
（3）优化目标：完成KV Cache压缩优化、推理解码优化、内存管理优化、算子融合优化，降低显存占用、减少推理时延；
（4）稳定性目标：长时间连续推理无闪退、无内存泄漏、生成精度无明显异常。
二、主要工作内容
2.1 硬件与软件环境搭建
搭建睿芯芯片串联开发环境，完成驱动适配、推理编译环境、调试工具链部署；完成Qwen3-8B模型权重转换、模型适配、硬件层算子兼容性调试，实现模型基础推理通跑。
2.2 KV Cache优化设计与实现
针对大模型推理过程中KV缓存占用高、访存频繁等痛点，开展KV Cache专项优化。主要包含：KV缓存量化压缩、动态缓存扩容与回收、缓存复用机制、片上缓存调度、闲置内存碎片整理；降低显存占用，减少Attention访存开销，提升解码阶段吞吐能力。
2.3 推理解码链路优化
针对模型解码串行延迟高的问题，优化生成解码流程。包含：预填充阶段加速、解码流水线并行、Attention计算优化、算子融合、层间数据零拷贝、推理调度策略优化；缩短首token延迟，提升生成阶段推理速度。
2.4 多芯片串联部署优化
结合睿芯芯片硬件特性，采用模型分层切分方式实现多芯片串联推理；优化芯片间数据传输链路，制定负载均衡策略，构建分布式KV缓存管理机制，解决单芯片显存不足、算力受限问题。
2.5 性能调优与压力测试
完成多轮迭代调优，对首包时延、生成速度、显存占用、CPU/NPU负载进行监控；开展长文本测试、多轮对话测试、长时间稳定性压力测试，验证优化效果，输出完整性能测试报告。
三、关键技术优化方案
3.1 KV Cache优化方案
（1）采用INT8量化压缩KV数据，降低缓存占用；
（2）设计动态滑动窗口缓存机制，自动回收无效KV缓存；
（3）优化缓存读写逻辑，减少片外内存访问频次；
（4）构建分布式KV缓存池，适配多芯片串联架构。
3.2 推理解码优化方案
（1）Prefill预计算加速，压缩首包生成时延；
（2）解码阶段流水并行，弱化串行等待耗时；
（3）融合归一化、激活、矩阵乘算子，降低计算开销；
（4）优化采样策略，在保证精度前提下提升生成速度。
